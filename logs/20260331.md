## 一、总体数据流

```
论文 PDF
   │
   ▼ [Stage 0] data_cleaner.py
   │  过滤保留 Abstract + Method 等核心章节
   │  输出: cleaned_output/{paper_name}.txt
   │
   ▼ [Stage 1] qwen_infer.py  ← 微调的 Qwen2.5-Coder-7B（15GB，AutoDL 服务器）
   │  输入: 清洗后论文文本（≤6000 字符截断）
   │  输出: qwen_outputs/{paper_name}.xml
   │         格式: <file name="core_model.py">...</file>
   │
   ▼ [Stage 2] build_xkg.py
   │  解析 Qwen XML 输出 → 结构化 xKG 知识图谱 JSON
   │  输出: xkg/{paper_name}.json
   │
   ▼ [Stage 3] validate_xkg.py
   │  subprocess 沙盒执行每个 Code Node
   │  失败 → LLM 修复 → 重试（最多 3 轮）→ 仍失败则剪枝
   │  输出: xkg/{paper_name}_validated.json
   │
   ▼ [Stage 4] Paper2Code 3_coding.py（待改造）
   │  Planning: 生成完整文件列表 + config.yaml
   │  Coding: xKG 验证节点作为 done_file_dict 锚点注入
   │           剩余文件（trainer/dataloader 等）由 LLM 生成
   │  输出: outputs/{paper_name}_repo/
   │
   ▼ [Stage 5] refine_repo.py（待新建）
   │  aider Repo Map 分析跨文件符号依赖
   │  修复 import 错误 / 接口不一致
   │
   ▼ [Stage 6] Paper2Code eval.py
      ref_free 或 ref_based 评估，输出 1-5 分
```

#### 设计决策：为什么放弃完整 xKG 流程

原始 xKG 论文（arXiv:2510.17795）设计了三步流程：
1. Technique Extraction（LLM 从论文提取技术节点树）
2. Code Modularization（FAISS 检索 → LLM re-rank → 改写为自包含 Code Node）
3. Knowledge Filtering（无代码节点剪枝）

**根本矛盾**：xKG 原始方案面向 42 篇参考论文 × 多文件仓库（每篇几百个文件），LLM re-rank 负责"从几百个文件里找对的那几个"。而我们的场景是 Qwen-7B 输出单个 `core_model.py`，连"选哪个文件"的问题都不存在。测试中发现 LLM re-rank 对单文件场景返回 None 导致全部节点被剪枝（0 个节点存活）。

**决策**：重写 `build_xkg.py` 为简化版，直接解析 Qwen XML 输出，保留 `CodeProvider` ABC 接口供日后多论文语料库扩展。

#### 当前实现（简化版）核心结构

```python
class CodeProvider(ABC):
    @abstractmethod
    def provide(self, paper_title: str, paper_text: str) -> dict[str, str]: ...

class QwenOutputProvider(CodeProvider):
    """解析 <file name="...">...</file> XML，无任何 LLM 调用"""

class CorpusRAGProvider(CodeProvider):
    """TODO：多论文语料库 RAG（完整 xKG 原始方案），目前 raise NotImplementedError"""
```

#### xKG JSON 输出格式

```json
{
  "paper_title": "AttentionCalibration",
  "abstract": "...",
  "techniques": [
    {
      "name": "Core Model",
      "type": "Methodology",
      "description": "Core algorithm implementation from core_model.py...",
      "components": [],
      "code": {
        "implementation": "import torch\n...",
        "documentation": "",
        "package": ["torch"]
      }
    }
  ]
}
```

`verified` 字段不出现在此阶段（`verified=None` 时省略），由 Stage 3 填充。

**自调试循环**（复现 xKG 论文 Section 2.2.2）：

```python
for each Code Node:
    for round in range(max_rounds + 1):   # round 0 = 首次不修复直接执行
        exit_code, stdout, stderr = subprocess.run(code, timeout=60)
        if exit_code == 0: break          # 可执行，保留
        if round == max_rounds: prune     # 耗尽重试，从 xKG 剪除
        code = llm_debug(code, stderr)    # 修复后进入下一轮
```

目标：初始 52.38% 可执行率 → 100%（来自 xKG 论文 Table 3）。

1. **输入侧**：`data_cleaner.py` 处理 90 篇论文的 `_cleaned.json` → `cleaned_output/*.txt`（信息密度高的论文核心文本）
