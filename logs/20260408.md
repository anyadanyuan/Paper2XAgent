## 二、技术设计

### 2.1 测试架构

```python
class TestContext:
    """测试上下文管理器，负责：
    - 创建隔离的测试目录（临时或固定）
    - 自动清理（临时模式）
    """
    
def test_stageN_xxx(ctx: TestContext, ..., mock_llm: bool) -> Path:
    """每个 Stage 的测试函数：
    1. 接收上游输出路径
    2. 执行当前 Stage 逻辑（真实或 Mock）
    3. 验证输出正确性
    4. 返回输出路径供下游使用
    """
```

**设计原则**：

- **最小依赖**：Stage 1-2 无需任何 API 密钥即可运行
- **可组合**：每个 Stage 可独立 skip，支持增量测试
- **隔离性**：默认使用临时目录，避免污染工作区
- **可检查性**：`--keep-outputs` 保留中间产物供调试

### 2.2 Mock 策略

| Stage | Mock 实现 | 真实实现 |
|-------|----------|----------|
| 0 | 直接写入 `train_dataset.json` 中的样本文本 | 同左（无需 Mock） |
| 1 | 直接写入 `train_dataset.json` 中的 XML | 真实调用 `qwen_infer.py` |
| 2 | 无需 Mock（直接调用 `build_xkg.py`） | 同左 |
| 3 | 标记所有节点为 `verified=True`，代码替换为固定模板 | 调用 `validate_xkg.py` |
| 4 | 手动创建最小 repo（config.yaml + 注入文件 + trainer.py） | 调用 `coding_inject.py` |
| 5 | 跳过（Mock 模式下无需修复） | 调用 `refine_repo.py` |
| 6 | 返回固定评分 `{"score": 4}` | 调用 `eval.py` |

**Mock 代码模板**：

```python
_MOCK_DEBUG_RESPONSE = """
import torch
import torch.nn as nn

class CoreModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer = nn.Linear(10, 10)

        return self.layer(x)
"""
```

## 六、已知问题与限制

### 6.1 Windows 控制台编码问题（已修复）

**问题**：原始代码中使用 Unicode 符号（✓、✅、⚠️、❌）导致 Windows PowerShell 报错：

```
UnicodeEncodeError: 'gbk' codec can't encode character '\u2713'
```

**解决方案**：将所有 Unicode 符号替换为 ASCII 等价物：

- `✓` → `[OK]`
- `✅` → `[PASS]`
- `⚠️` → `[WARN]`
- `❌` → `[ERROR]`

### 6.2 测试数据依赖

**限制**：测试依赖 `train_dataset.json` 的第一条样本数据。

- 在测试启动时明确检查 `_TRAIN_DATASET` 路径
- 提供清晰的错误提示
