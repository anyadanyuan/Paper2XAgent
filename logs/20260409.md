## 一、问题回顾：pip 镜像源超时与 JSON 解析错误

### 1.1 问题 1：清华镜像源读取超时

**错误信息**：
```
pip._vendor.urllib3.exceptions.ReadTimeoutError: 
HTTPSConnectionPool(host='pypi.tuna.tsinghua.edu.cn', port=443): Read timed out.
```

**分析**：
- 清华源连接成功（HTTPS 443 端口可达）
- 开始下载包文件
- 但在读取数据过程中超时（默认读取超时时间太短）

**失败的尝试方案**：
```bash
# 增加超时时间（依然超时）
pip install --upgrade "openai>=1.30.0" \
    -i https://pypi.tuna.tsinghua.edu.cn/simple \
    --timeout 300
```

---

### 1.2 问题 2：JSON 解析错误

**错误信息**：
```
json.decoder.JSONDecodeError: Unterminated string starting at: 
line 1 column 2981617 (char 2981616)
```

**根本原因**：
- 镜像源返回的包索引 JSON 在传输过程中被截断
- pip 收到的 JSON 响应不完整/损坏
- 可能是镜像源本身的问题或网络不稳定

---

## 二、成功的解决方案

### 2.1 使用预编译轮子 (whl) 安装

**关键命令**：
```bash
# 在 llama 环境下执行（重要！服务器有 CUDA 驱动）
conda activate llama

# 跳过依赖解析，直接安装预编译包
pip install --upgrade openai \
    --prefer-binary \
    --no-deps \
    -i https://pypi.tuna.tsinghua.edu.cn/simple

# 补充安装依赖
pip install httpx anyio distro tqdm typing-extensions \
    -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**关键参数说明**：
- `--prefer-binary`：优先使用预编译的 whl 文件，避免源码编译
- `--no-deps`：跳过依赖解析，避免 JSON 解析错误
- 后续手动安装必要依赖

**成功结果**：
```
Successfully installed openai-1.55.3
[OK] OpenAI import successful
```

---

### 2.2 环境要求：必须在 llama 环境运行

- 服务器已安装 CUDA 驱动
