## 一、评测目标

**核心问题**：验证微调后的 Qwen2.5-Coder-7B-SFT 相比基座 Qwen2.5-Coder-7B-Instruct，在生成"完整可运行且忠于原论文算法"的代码方面的能力提升。

**最终目标**：生成完整可运行的、忠于原论文算法的核心模型架构代码（PyTorch）。

**评测原则**：
- 尽量减少 LLM 作为评分器的使用（避免主观性、降低成本）
- 可调用 LLM 进行辅助性提取/结构化，但**最终评分由规则/执行验证完成**
- 评估应直接对比两个模型的输出，使用相同的评测脚本和指标
- **评测流程独立**，不依赖 paper2XAgent pipeline 的任何阶段（不调用 build_xkg、validate_xkg、coding_inject 等）

## 二、评测指标体系（完整设计）

基于"可运行性 × 论文忠实度"双维度，共 5 个核心指标 + 2 个辅助指标。

### 2.1 第一维度：可运行性（Execution）

| 指标 | 缩写 | 定义 | 测量方式 | LLM 参与 |
|------|------|------|---------|---------|
| **可执行率** | XPR | 生成代码在沙盒中直接执行成功（exit_code=0）的比例 | `subprocess.run()` 执行 Python 代码 | ❌ 无 |
| **独立运行性** | SAR | 在**隔离环境**（无预注入变量/上下文）中执行成功的比例 | 禁止任何外部变量注入，纯执行代码字符串 | ❌ 无 |
| **语法正确率** | SYR | 代码能通过 `ast.parse()` 解析为有效 Python AST 的比例 | `ast.parse(code)` 不抛出 SyntaxError | ❌ 无 |

**关键区别**：
- XPR 允许代码依赖运行时环境中已导入的模块（如我们预装了 PyTorch）
- SAR 要求代码自身包含所有必要 import，且不能引用未定义的变量（如 `config` 对象）
- SYR 仅检查语法结构，不执行（最快、最基础的过滤）

**预期区分度**：微调模型 > 基座模型（微调学会了完整可运行代码的生成模式）

### 2.2 第二维度：论文忠实度（Fidelity）

| 指标 | 缩写 | 定义 | 测量方式 | LLM 参与 |
|------|------|------|---------|---------|
| **论文组件覆盖率** | PCR | 生成代码包含论文中描述的核心技术组件的比例 | LLM 从论文提取组件清单 → 规则匹配代码 | ⚠️ 仅提取清单 |
| **参考代码语义相似度** | CodeBERTScore | 生成代码与参考代码（GitHub 蒸馏）的语义相似度 | CodeBERT embedding 计算 F1 | ❌ 无 |
| **API 调用对齐率** | AAR | 生成代码使用的 PyTorch API 与参考代码的 Jaccard 相似度 | 正则提取 API 调用，计算集合交集 | ❌ 无 |

**PCR 详细设计**（最关键的创新指标）：

```
Step 1（一次性，可用 LLM）: 论文 → 结构化组件清单
  输入: 论文文本（Abstract + Method）
  输出: {
    "required_modules": ["nn.MultiheadAttention", "nn.LayerNorm", "nn.Dropout"],
    "required_patterns": ["residual_connection", "positional_encoding", "softmax_attention"],
    "required_functions": ["forward", "__init__"],
    "expected_dims": ["hidden_size", "num_heads"]
  }

Step 2（规则评分，无 LLM）: 代码 → 组件覆盖检查
  - required_modules: 检查字符串是否包含每个模块名
  - required_patterns: 用正则匹配结构模式（如残差连接 x + self.layer(x)）
  - required_functions: 检查是否定义了必要的方法
  - expected_dims: 检查参数化配置能力

PCR = 满足的检查项数 / 总检查项数
```

**AAR 详细设计**：

```
提取: 从代码中提取 torch.*, nn.*, F.* 调用
  例: "attn_weights = F.softmax(attn_weights, dim=-1)" → {F.softmax}

对齐: Jaccard(pred_apis, ref_apis) = |pred ∩ ref| / |ref|

变体（无参考代码时）:
  论文→代码 AAR: 从论文文本提取隐含 API（如"layer normalization"→nn.LayerNorm），检查代码中是否出现
```

### 2.3 辅助指标

| 指标 | 缩写 | 定义 | 用途 |
|------|------|------|------|
| **调试轮次需求** | DRR | 平均需要 LLM 调试多少轮才能使代码可运行 | 间接反映代码质量（越低越好） |
| **代码长度比** | CLR | 生成代码长度 / 参考代码长度 | 检测过度生成或过于简略 |

# evaluation/evaluators/fidelity_eval.py

class FidelityEvaluator:
    """论文忠实度指标评估器"""

    def __init__(self, ref_code: str = None, paper_text: str = None):
        self.ref_code = ref_code
        self.paper_text = paper_text
        self.component_list = None  # 缓存 PCR 清单

    def evaluate(self, pred_code: str) -> dict:
        """
        返回:
        {
            "pcr": float,              # 0-1
            "aar": float,              # 0-1
            "codebertscore": float,    # 0-1
            "component_hits": list,    # 哪些组件被命中
            "missing_components": list,# 哪些组件缺失
        }
        """
        pass

    def compute_pcr(self, pred_code: str, paper_text: str) -> float:
        """论文组件覆盖率"""
        if self.component_list is None:
            self.component_list = self._extract_components_from_paper(paper_text)

        hits = 0
        total = 0
        for module in self.component_list.get("required_modules", []):
            if module in pred_code:
                hits += 1
            total += 1

        for pattern in self.component_list.get("required_patterns", []):
            if self._match_pattern(pred_code, pattern):
                hits += 1
            total += 1

        return hits / total if total > 0 else 0.0

    def compute_aar(self, pred_code: str, ref_code: str) -> float:
        """API 调用对齐率"""
        pred_apis = self._extract_apis(pred_code)
        ref_apis = self._extract_apis(ref_code)
        if not ref_apis:
            return 0.0
        return len(pred_apis & ref_apis) / len(ref_apis)

    def compute_codebertscore(self, pred_code: str, ref_code: str) -> float:
        """CodeBERTScore"""
        from code_bert_score import score
        _, _, f1, _ = score([pred_code], [ref_code], lang='python')
        return f1.item()

    def _extract_components_from_paper(self, paper_text: str) -> dict:

        用 LLM 从论文提取组件清单。

        """

    def _extract_apis(self, code: str) -> set:
        """正则提取 PyTorch API 调用"""
        import re
        apis = set()
        patterns = [
            r'torch\.([a-zA-Z_][a-zA-Z0-9_]*)',
            r'nn\.([a-zA-Z_][a-zA-Z0-9_]*)',
            r'F\.([a-zA-Z_][a-zA-Z0-9_]*)',
            r'nn\.functional\.([a-zA-Z_][a-zA-Z0-9_]*)',
        ]
        for pat in patterns:
            apis.update(re.findall(pat, code))
        return apis

    def _match_pattern(self, code: str, pattern_name: str) -> bool:
        """匹配代码结构模式"""
        patterns = {
            "residual_connection": r'\w+\s*\+\s*\w+\([^)]*\)',
            "positional_encoding": r'pos_emb|positional|PositionalEncoding',
            "softmax_attention": r'F\.softmax.*attn|softmax.*dim\s*=\s*-1',
            # ... 可扩展
        }
        if pattern_name in patterns:
            return bool(re.search(patterns[pattern_name], code))
        return False

```python
# evaluation/evaluate.py（主脚本）

class ModelEvaluator:
    """端到端评测器"""

    def __init__(self, model_path: str, use_api: bool = False):
        self.model = ModelLoader(model_path, use_api)
        self.execution_eval = ExecutionEvaluator()
        self.fidelity_eval = FidelityEvaluator()

    def evaluate_dataset(self, dataset: list[dict], output_dir: str) -> dict:
        """
        对数据集进行完整评测。
        dataset: [{"paper_text": str, "ref_code": str, "paper_id": str}, ...]
        """
        results = []
        for sample in dataset:
            # 1. 模型推理
            pred_xml = self.model.generate(sample["paper_text"])
            pred_code = self._extract_code(pred_xml)

            # 2. 执行评估
            exec_result = self.execution_eval.evaluate(pred_code)

            # 3. 忠实度评估
            self.fidelity_eval.ref_code = sample["ref_code"]
            self.fidelity_eval.paper_text = sample["paper_text"]
            fid_result = self.fidelity_eval.evaluate(pred_code)

            results.append({
                "paper_id": sample.get("paper_id", ""),
                "pred_code": pred_code,
                "exec": exec_result,
                "fidelity": fid_result,
            })

        # 4. 汇总统计
        summary = self._aggregate(results)
        self._save_results(results, summary, output_dir)
        return summary

    def _extract_code(self, xml_output: str) -> str:
        """从模型输出中提取代码，兼容基座和微调模型"""
        import re
        matches = re.findall(r'<file name="([^"]+)">(.*?)</file>', xml_output, re.DOTALL)
        if matches:
            return matches[0][1].strip()
        code_blocks = re.findall(r'```python\n(.*?)```', xml_output, re.DOTALL)
        if code_blocks:
            return code_blocks[0].strip()
        return xml_output.strip()

    def _aggregate(self, results: list) -> dict:
        """汇总统计"""
        n = len(results)
        return {
            "xpr": sum(1 for r in results if r["exec"]["xpr"]) / n,
            "sar": sum(1 for r in results if r["exec"]["sar"]) / n,
            "syr": sum(1 for r in results if r["exec"]["syr"]) / n,
            "pcr_avg": sum(r["fidelity"]["pcr"] for r in results) / n,
            "aar_avg": sum(r["fidelity"]["aar"] for r in results) / n,
            "cbs_avg": sum(r["fidelity"]["codebertscore"] for r in results) / n,
            "total_samples": n,
        }
```

## 六、关键设计决策

### 6.1 为什么评测独立于主项目？

- 本评测用于**另一个项目**的对比验证，不是 paper2XAgent pipeline 的一部分
- 避免耦合：不调用 `build_xkg.py`、`validate_xkg.py`、`coding_inject.py` 等 pipeline 模块
- 最小复用：仅从 `validate_xkg.py` **复制** `run_in_sandbox()` 函数，不导入整个模块
- 独立演进：评测指标和主项目可以各自迭代，互不干扰

### 6.2 为什么不用训练集当测试集？

- **数据泄露**：微调模型在训练集上见过输入输出对，评测结果会虚高
- **无法反映泛化能力**：我们需要的是模型对**未见过的论文**的生成能力
- **学术规范**：任何严肃的模型对比都必须使用独立的测试集
- **测试集构建**：后续单独进行（见 Phase 5），当前脚本只预留接口

### 6.3 为什么不用完整 xKG 流程评估基座模型？

基座模型输出格式不稳定，经常：
- 不输出 `<file>` 标签
- 输出自然语言解释 + 代码混合
- 代码不完整（缺少 import、class 定义）

如果强制走 `build_xkg.py` → `validate_xkg.py` 的完整流程：
- `QwenOutputProvider` 的 `<file>` 正则可能失效
- 基座模型的"伪代码"会被错误解析为 `core_model.py`
- 评估结果失真

**解决方案**：评估层直接操作代码字符串，xKG 是 pipeline 产物而非评估前提。

### 6.4 PCR 中 LLM 的使用策略

| 阶段 | LLM 角色 | 调用频率 |
|------|---------|---------|
| 组件清单提取 | 从论文文本提取结构化要求 | **每篇论文 1 次**，结果缓存 |
| 代码评分 | 无，纯规则匹配 | 零次 |

**成本估算**：50 篇测试论文 × 1 次 API 调用 ≈ $0.05（使用 gpt-4.1-mini）

### 6.5 指标权重建议（最终评分公式）

```python
def overall_score(metrics: dict) -> float:
    """
    综合评分：可运行性 50% + 论文忠实度 50%
    """
    execution = (
        0.4 * metrics["xpr"] +
        0.4 * metrics["sar"] +
        0.2 * metrics["syr"]
    )
    fidelity = (
        0.5 * metrics["pcr"] +
        0.3 * metrics["codebertscore"] +
        0.2 * metrics["aar"]
    )
    return 0.5 * execution + 0.5 * fidelity
```

这个权重可以根据 Phase 6 的实际结果调整。

### 6.6 沙盒执行的安全性与隔离

从 `validate_xkg.py` **复制** `subprocess.run()` 方案：
- 使用临时文件 `tempfile.NamedTemporaryFile`
- 设置超时 `timeout=60`
- 捕获 stdout/stderr，不污染主进程
- 环境变量隔离 `PYTHONDONTWRITEBYTECODE=1`

**注意**：基座模型可能生成危险代码（如 `os.system("rm -rf /")`），虽然概率极低，但沙盒隔离是必需的。

## 八、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 基座模型输出格式过于混乱，无法提取有效代码 | SAR/XPR 计算失败 | `code_parser.py` 增加多种 fallback 策略 |
| CodeBERTScore 计算耗时过长 | 测试集评估慢 | 批量计算，使用 GPU 加速 |
| PCR 组件清单提取不准确 | PCR 评分失真 | 人工抽检 10% 样本，校准提取 prompt |
| 沙盒执行安全风险 | 恶意代码破坏环境 | 使用 Docker 隔离或限制系统调用 |
| 测试集样本量不足 | 统计不显著 | 目标 20-50 篇，可扩展到更多 |
| 测试集构建耗时 | 阻塞评测 | 与 Phase 1-4 并行进行（评测脚本不依赖具体数据） |
